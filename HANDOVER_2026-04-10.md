# Handover Addendum (2026-04-10)

## Scope
- Replace salary-band display and calculation defaults with position average annual salary.
- Restrict actual salary viewing/editing to explicitly authorized users.
- Verify the live Dockerized database reflects the new schema names.

## Delivered Changes

### 1) Salary model redesign
- System default pay basis is now `positionAverageAnnualSalary`.
- Project-level actual pay is now `actualAnnualSalaryOverride`.
- Calculation priority is:
  - `actualAnnualSalaryOverride`
  - `positionAverageAnnualSalary`

### 2) Authorization redesign
- User entity now has `canManageActualSalary`.
- Effective permission rule:
  - `ADMIN` always allowed
  - otherwise `canManageActualSalary = true` required
- Frontend settings page exposes this as a per-user toggle.

### 3) Backend/API changes
