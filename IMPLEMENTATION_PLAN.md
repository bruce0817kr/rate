## Phase 3: 팀원 관리 (Personnel CRUD)

### 3.1 DB 스키마 업데이트 (백엔드)
```typescript
// personnel.entity.ts 업데이트
highestEducation: string;        // 학위: "박사", "석사", "학사"
educationMajor: string;          // 전공: "컴퓨터공학"
actualSalary?: number;           // 실제 연봉 (선택, 미입력 시 salaryBand 사용)
```

### 3.2 팀원 목록 페이지
- [ ] 테이블: 성명, 직급, 소속팀, 참여율(%), 참여 사업수, 상태
- [ ] 상태: 정상(초록), 주의(노랑), 위험(빨강 - 100%초과)
- [ ] 정렬/필터링

### 3.3 팀원 상세/편집 페이지
- [ ] **기본 정보**: 사번, 성명, 직급, 소속팀, 재직상태
- [ ] **학력 정보**: 최종학위, 취득연도, 전공
- [ ] **급여 정보**: 
  - 직급별 기본 급여대역표 (설정에서 관리)
  - "직급 선택 → 급여대역 자동 입력" 기능
  - 실제 급여 직접 입력 가능 (선택)
- [ ] **참여율 현황**: 
  - 현재 참여율 게이지 (100% 기준)
  - 참여중인 사업 목록
  - 역할별 참여율 (3책/5공 규칙 표시)

### 3.4 참여율 입력/수정
- [ ] 모달/다이얼로그: 사업 선택 → 역할 선택 → 참여율 입력
- [ ] **유효성 검증**: 
  - 0-100% 범위
  - 3책/5공 규칙
  - 개인 총 참여율 100% 초과 시 경고

## API Endpoints to Add

### Personnel (확장)
- `PATCH /personnel/:id` — 전체 필드 업데이트
- `POST /personnel/:id/education` — 학력 정보 별도 관리
- `GET /personnel/with-participation` — 참여율 포함 조회

### Team
- `GET /teams` — 팀 목록
- `POST /teams` — 팀 생성
- `PATCH /teams/:id` — 팀 수정
- `DELETE /teams/:id` — 팀 삭제

### Participation Monitoring (개선)
- `GET /participation-monitoring/individual` — **개인별** 참여율 현황 (NEW!)
- `GET /participation-monitoring/individual/:id` — 특정 개인 상세
- `GET /participation-monitoring/team/:teamName` — 특정 팀 현황
