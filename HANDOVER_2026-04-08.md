- Remove mock-first data flow.

  - Implemented in:

    - `backend/src/modules/upload/upload.service.ts`

### 3) Why names looked like `...팀원1`
- Root cause: seed-generated mock naming pattern (`구성원1`, `팀원1` style) in `backend/src/seed/personnel.seed.ts`.
- To stop auto-mock flow on startup:
  - Removed `SeedModule` from runtime app imports in `backend/src/app.module.ts`.

### 4) Mock cleanup execution
- Added API:
  - `POST /api/personnel/purge-mock`
  - Implemented in:
    - `backend/src/modules/personnel/personnel.controller.ts`
    - `backend/src/modules/personnel/personnel.service.ts`
- Cleanup rule:
  - Name regex: `(팀원|구성원)[0-9]+$`
  - EmployeeId regex: `^EMP[0-9]{4,}$`
- Execution result on running environment:
  - `deletedCount = 80`
  - Verified remaining mock records: `0`
