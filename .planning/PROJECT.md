# GTP 참여율 관리 시스템

## What This Is

경기테크노파크의 팀, 팀원, 사업, 참여율, 인건비를 통합 관리하는 내부 운영 시스템이다.
관리자는 사업별 참여 인력과 참여율, 계산 인건비, 조직 현황을 한 화면에서 확인하고 조정할 수 있어야 한다.
현재 마일스톤은 대시보드 의미 정리, 깨진 한글 복구, 차트 및 표기 서식 개선에 집중한다.

## Core Value

운영자가 깨지지 않은 한글 UI와 신뢰할 수 있는 참여율·인건비 수치로 사업 현황을 즉시 판단할 수 있어야 한다.

## Requirements

### Validated

- [x] 팀, 팀원, 사업 CRUD 화면이 존재한다.
- [x] 사업 상세에서 참여 인력과 참여율을 관리할 수 있다.
- [x] CSV 업로드 및 관리자 기능이 시스템에 포함되어 있다.

### Active

- [ ] 대시보드와 차트의 모든 한글 텍스트가 정상 표기된다.
- [ ] 대시보드 지표와 차트가 실제 참여율/인건비 의미와 맞게 동작한다.
- [ ] 팀원/팀/사업 관련 화면의 퍼센트·금액·정렬 서식이 일관된다.
- [ ] 빌드와 도커 재기동까지 포함해 변경 사항이 실제 반영된다.

### Out of Scope

- 모바일 앱 별도 개발: 현재는 웹 운영 화면 안정화가 우선이다.
- 신규 외부 연동 추가: 현재 마일스톤은 기존 데이터/화면 정합성 보정이 목적이다.

## Context

- 실제 앱 작업 트리는 `saas-project` 하위의 `frontend`, `backend`이다.
- 프런트엔드에는 아직 한글 인코딩이 깨진 문자열이 남아 있다.
- Recharts는 이미 설치되어 있고 대시보드 차트 컴포넌트가 존재한다.
- 최근 사용자 요청은 대시보드 지표 의미 정리, 차트 개선, 서식 통일, 빌드/도커 반영까지 포함한다.

## Constraints

- **Tech stack**: React + TypeScript + NestJS 구조를 유지해야 한다.
- **Brownfield**: 기존 사용자 데이터와 현재 CRUD 흐름을 깨면 안 된다.
- **UI language**: 사용자 노출 텍스트는 한국어 기준으로 정리해야 한다.
- **Verification**: 프런트 빌드와 도커 재기동까지 확인해야 완료로 본다.

## Key Decisions

| Decision | Rationale | Outcome |
|----------|-----------|---------|
| 현재 마일스톤을 대시보드/차트/서식 정리로 한정 | 최근 사용자 요청이 이 축에 집중되어 있음 | Pending |
