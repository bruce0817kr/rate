# 급여 밴드 제거 및 직급별 평균 연봉 체계 전환 설계

## 목표

현재 시스템의 `급여 밴드` 개념을 제거하고, 전사 공통 기준값을 `직급별 평균 연봉` 단일 숫자값으로 통일한다.

추가로 프로젝트 참여 인력별로만 `실제 연봉 override`를 둘 수 있게 하며, 이 값은 제한된 사용자만 조회/수정할 수 있어야 한다.

## 확정된 정책

### 전사 급여 기준

- 급여 밴드는 폐기한다.
- 직급별 기준값은 `직급별 평균 연봉` 숫자 하나만 관리한다.
- 일반 사용자가 보는 급여 기준값은 항상 직급별 평균 연봉이다.

### 프로젝트 계산 정책

- 프로젝트 인건비 계산 기본값은 직급별 평균 연봉이다.
- 프로젝트 참여 정보에 `실제 연봉 override`가 있으면 계산 시 override를 우선 사용한다.
- override가 없으면 직급별 평균 연봉을 사용한다.

### 실제 연봉 권한 정책

- `ADMIN`은 항상 실제 연봉 override를 조회/수정할 수 있다.
- 그 외에는 관리자가 별도로 지정한 `HR_GENERAL_AFFAIRS`, `STRATEGY_PLANNING` 소속 직원만 실제 연봉 override를 조회/수정할 수 있다.
- 역할만으로 자동 허용하지 않는다.
- 이를 위해 사용자 엔티티에 별도 권한 플래그를 둔다.

## 권장 데이터 모델

### 사용자

기존 `role`과 별도로 실제 연봉 권한을 명시하는 필드를 추가한다.

- `canManageActualSalary: boolean`

의도:

- 역할과 민감정보 권한을 분리한다.
- 전략기획팀 전체를 열지 않고 담당자 단위로 통제한다.

### 직급 기준 연봉 설정

기존 `salary_bands`는 의미와 구조를 모두 교체한다.

권장 모델:

- 테이블명: `position_salary_settings`
- 주요 필드:
  - `id`
  - `position`
  - `averageAnnualSalary`
  - `isActive`
  - `createdAt`
  - `updatedAt`

기존 `minAmount`, `maxAmount`는 제거한다.

### 인력 정보

기존 `personnel.salaryBand`는 제거한다.

인력 레코드에는 직급만 유지하고, 평균 연봉은 직급 설정에서 조회한다.

의도:

- 사람 데이터에 밴드 문자열을 중복 저장하지 않는다.
- 직급 기준 변경 시 전사 기준값을 한 군데에서만 관리한다.

### 프로젝트 참여 정보

기존 `project_personnel.annualSalary`는 의미를 명확히 바꾼다.

권장 필드명:

- `actualAnnualSalaryOverride: number | null`

기존 필드명을 즉시 바꾸기 어렵다면 1차 마이그레이션에서는 내부 컬럼명을 유지할 수 있다. 다만 API DTO, 서비스 변수명, 프런트 타입은 모두 `actualAnnualSalaryOverride`로 정리한다.

## API 설계

### 설정 API

기존 `salary-bands` API는 `position-salary-settings` 성격으로 바꾼다.

응답 예시:

```json
{
  "id": "uuid",
  "position": "선임연구원",
  "averageAnnualSalary": 62000000,
  "isActive": true
}
```

### 인력 API

인력 응답에서는 더 이상 `salaryBand`를 내려주지 않는다.

필요 시 표시용 필드로 아래 중 하나를 제공한다.

- `positionAverageAnnualSalary`
- 또는 프런트가 설정 API를 참조해 계산

권장은 서버 응답에 `positionAverageAnnualSalary`를 포함해 프런트 중복 로직을 줄이는 방식이다.

### 프로젝트 참여 API

권한 없는 사용자는 `actualAnnualSalaryOverride`를 응답에서 볼 수 없어야 한다.

정책:

- 권한 있음: `actualAnnualSalaryOverride` 반환
- 권한 없음: `actualAnnualSalaryOverride: null`

중요:

- 프런트에서 숨기는 것만으로는 불충분하다.
- 백엔드에서 응답 마스킹을 강제해야 한다.

### 계산 API 및 집계

계산에 사용하는 기준값은 다음 우선순위를 따른다.

1. `actualAnnualSalaryOverride`
2. `positionAverageAnnualSalary`

이 우선순위는 대시보드, 프로젝트 상세, 참여 인건비 계산, 저장 로직에서 동일해야 한다.

## 화면 설계

### 설정 화면

기존 `급여 밴드 관리`를 `직급별 평균 연봉 관리`로 교체한다.

변경점:

- `min/max` 입력 제거
- `평균 연봉(원)` 단일 입력 사용
- 문구, 버튼, 토스트, 타입명을 평균 연봉 기준으로 통일

### 인력 상세 / 목록

기존 급여 밴드 표시는 제거하고 `직급별 평균 연봉`만 표시한다.

일반 사용자도 평균 연봉은 볼 수 있다.

### 프로젝트 상세

모든 참여 인력은 기본 계산값으로 평균 연봉을 사용한다.

권한 없는 사용자:

- 평균 연봉 기준 계산값만 확인 가능
- 실제 연봉 입력 필드는 보이지 않음

권한 있는 사용자:

- 평균 연봉 기준값을 확인 가능
- `실제 연봉 override` 입력/수정 가능
- 계산 결과가 override 적용인지 명확히 표시

권장 UI:

- `기준 평균 연봉`
- `실제 연봉 override`
- `적용 연봉`

세 필드를 분리해 보여 주되, 권한 없는 경우에는 `실제 연봉 override`를 렌더링하지 않는다.

## 업로드 및 템플릿 설계

인력 업로드 템플릿에서 `salaryBand` 컬럼을 제거한다.

선택지:

- 인력 업로드는 직급만 받는다.
- 직급별 평균 연봉은 설정 화면 또는 별도 직급 설정 업로드에서 관리한다.

권장:

- 인력 CSV는 직급만 유지
- 직급 기준 연봉은 설정 화면에서 별도 관리

이 방식이 데이터 소유권을 가장 분명하게 만든다.

## 마이그레이션 전략

### 1단계. 스키마 추가

- 사용자에 `canManageActualSalary` 추가
- 직급별 평균 연봉 설정 테이블 추가
- 프로젝트 참여 override 필드 명확화

### 2단계. 데이터 이전

- 기존 `salary_bands.position`을 새 설정 테이블로 이전
- `minAmount`, `maxAmount`는 midpoint 계산 후 `averageAnnualSalary`로 변환
- 기존 인력의 `salaryBand`는 이후 참조하지 않도록 정리

### 3단계. 백엔드 전환

- DTO, 엔티티, 서비스, 권한 체크, 응답 마스킹 전환
- 계산 서비스의 밴드 midpoint 파싱 제거

### 4단계. 프런트 전환

- 설정, 인력, 프로젝트 화면을 평균 연봉 기준으로 교체
- `salaryBand` 관련 타입, 함수, 라벨 제거

### 5단계. 정리

- 구 테이블/구 필드/구 파서 제거
- 관련 문서 및 템플릿 업데이트

## 오류 처리 및 예외

- 직급에 대응하는 평균 연봉 설정이 없으면 프로젝트 참여 추가/계산 시 명시 오류를 반환한다.
- 권한 없는 사용자가 실제 연봉 필드를 수정하려 하면 403을 반환한다.
- 실제 연봉 override는 0 이하 값을 허용하지 않는다.
- 직급 변경 시 해당 직급 평균 연봉 설정이 없으면 저장 단계에서 경고 또는 차단한다.

## 테스트 전략

### 백엔드

- 직급 평균 연봉 기반 계산 테스트
- override 우선 적용 테스트
- 권한 없는 사용자 응답 마스킹 테스트
- 권한 있는 지정 사용자 응답 허용 테스트
- midpoint 마이그레이션 변환 테스트

### 프런트

- 설정 화면 단일 평균 연봉 입력 테스트
- 일반 사용자 화면에서 override 비노출 테스트
- 권한 사용자 화면에서 override 노출 테스트
- 프로젝트 상세 계산 우선순위 표시 테스트

### 회귀 검증

- 대시보드 인건비 합계
- 프로젝트 상세 저장/재조회
- 인력 업로드
- 설정 수정 후 계산 반영

## 구현 원칙

- 밴드 문자열 파싱 로직은 최종적으로 제거한다.
- 평균 연봉 표시와 실제 연봉 override 표시를 혼동하지 않도록 필드명과 UI 문구를 분리한다.
- 민감정보는 백엔드에서 차단하고 프런트는 그 결과를 소비만 한다.
- 계산 우선순위는 한 곳의 공용 함수/서비스에 모은다.

## 비목표

- 개인별 실제 연봉을 인력 기본 프로필에 저장하지 않는다.
- 전략기획팀 전체에 자동 권한을 부여하지 않는다.
- 급여 밴드와 평균 연봉을 병행 운영하지 않는다.

## 추천 구현 순서

1. 사용자 권한 플래그 추가
2. 직급별 평균 연봉 설정 모델 및 마이그레이션 추가
3. 계산 서비스 공용 기준값 로직 교체
4. 프로젝트 참여 API의 실제 연봉 필드 마스킹 적용
5. 설정 화면 교체
6. 프로젝트 상세 화면 교체
7. 인력 화면 및 업로드 템플릿 정리
8. 구 밴드 코드 제거
